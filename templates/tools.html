Stuctures Python

For a collection of strings in Python, the list is the standard and most versatile choice. Choosing the right structure depends on how you plan to access or modify that data:

  1. List (Best for General Use) Use a Python list if you need an ordered sequence of strings that you can add to, remove from, or change.
    Pros: Built-in, highly flexible, supports duplicates, and maintains insertion order.
    Example: names = ["Alice", "Bob", "Charlie"]
  2. Dictionary (dict) (Best for Lookups) Use a Python dictionary if each string is associated with a specific label or if you need to find a value instantly without searching the whole collection.
    Pros: Fast lookups by key; ideal for mapping unique identifiers to data.
    Example: user_emails = {"Alice": "alice@eg.com", "Bob": "bob@eg.com"}
  3. Set (Best for Uniqueness & Search) Use a Python set if you only care whether a string exists in the collection and want to ensure there are no duplicates.
    Pros: Extremely fast "membership testing" (checking if "Alice" is in the set) compared to a list.
    Example: unique_tags = {"python", "coding", "dev"}
  4. Tuple (Best for Fixed Data) Use a Python tuple if the list of strings will never change after it is created.
    Pros: Slightly more memory-efficient than lists and can be used as keys in dictionaries because they are immutable.
    Example: coordinates = ("North", "West")
  5. Array (Rarely used for strings) The Python array module or NumPy arrays are primarily optimized for numerical data.
    Note: While you can store strings in NumPy arrays, they are rarely used for simple string lists unless you are performing massive scientific computations or matrix operations.

Pandas Joins


import pandas as pd

# Creating dataframe a
a = pd.DataFrame()
d = {'id': [1, 2, 10, 12],
     'val1': ['a', 'b', 'c', 'd']}
a = pd.DataFrame(d)

# Creating dataframe b
b = pd.DataFrame()
d = {'id': [1, 2, 9, 8],
     'val1': ['p', 'q', 'r', 's']}
b = pd.DataFrame(d)
Inner Join

Inner join is the most common type of join you’ll be working with.

It returns a Dataframe with only those rows that have common characteristics. This is similar to the intersection of two sets..


# inner join
df = pd.merge(
  a, b,
  on='id',
  how='inner'
)
Left Outer Join

SQL joins require defined positions for the tables being joined, typically referred to as the left and right tables.

With a left outer join, all the records from the first Dataframe will be displayed, irrespective of whether the keys in the first Dataframe can be found in the second Dataframe. Whereas, for the second Dataframe, only the records with the keys in the second Dataframe that can be found in the first Dataframe will be displayed.


# left outer join
df = pd.merge(
  a, b,
  on='id',
  how='left'
)
Right Outer Join

The order determines which table serves as the primary base (left) and which is matched against it (right).

For a right join, all the records from the second Dataframe will be displayed. However, only the records with the keys in the first Dataframe that can be found in the second Dataframe will be displayed.


# right outer join
df = pd.merge(
  a, b,
  on='id',
  how='right'
)
Outer Join

This is also called Full Outer Join or simply Outer Join.

A full outer join returns all the rows from the left Dataframe, and all the rows from the right Dataframe, and matches up rows where possible, with NaNs elsewhere. But if the Dataframe is complete, then we get the same output.


# full outer join
df = pd.merge(
  a, b,
  on='id',
  how='outer'
)
Index Join

To merge the Dataframe on indices pass the left_index and right_index arguments as True i.e. both the Dataframes are merged on an index using default Inner Join.

To merge the Dataframe on indices pass the left_index and right_index arguments as True i.e. both the Dataframes are merged on an index using default Inner Join.


# index join
df = pd.merge(
  a, b,
  left_index=True,
  right_index=True
)
