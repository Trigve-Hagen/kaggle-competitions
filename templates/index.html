Logistic Regression

Cross-entropy is a loss function used in machine learning to measure the performance of classification models by calculating the difference between predicted probabilities and actual labels. Logits are the raw, unnormalized, real-valued scores produced by the final linear layer of a neural network before applying an activation function like softmax. Softmax converts raw model outputs (logits) into probabilities, while cross-entropy measures the difference between these probabilities and true labels. Together, they provide stable, large gradients during training—avoiding vanishing gradients, especially at the start—and form a convex loss function for efficient optimization. A gradient is a vector of partial derivatives that points in the direction of the steepest ascent of a loss function. Logarithms The log() function, or logarithm tells us what the exponent is. 2x=8 becomes log28=x which is log28=3 It answers the question: "To what power must a given base be raised to obtain a specific number?"

Consider the equation 103=1000. In this case, the base b is 10, the exponent y is 3, and the result x is 1000. The logarithmic form of this equation is log10(1000)=3. This means "the power to which 10 must be raised to get 1000 is 3."

Key Elements:
Base: The number that is being raised to a power. Common bases include 10 common logarithm, often written as log(x) without a subscript and e natural logarithm, written as ln(x).
Argument: The number for which the logarithm is being calculated. This number must always be positive.
Result: The exponent to which the base must be raised to equal the argument.

Example: log2(8).
This asks: "To what power must 2 be raised to get 8?"
We know that 2^1 = 2, We know that 2^2 = 4, and We know that 2^3 = 8.
Therefore log2(8)=3.

The logarithm in the cross-entropy loss function is used to heavily penalize confident, wrong predictions, convert multiplicative probabilities into additive losses for easier optimization, and provide numerical stability. It turns small errors into manageable losses and large errors into massive losses, driving better classification performance.

Drastic Penalty for Wrong Predictions: If the true label y = 1 and the model predicts a low probability ŷ = 0.01 (high confidence, wrong), -log(0.01) becomes a large number. Conversely, if y = 1 and ŷ = 0.99 (high confidence, right), -log(0.99) is very low.

Entropy

"The negative sum of probabilities multiplied by their log."

E=-∑P(X)*logP(X)

The summation represents the combining of the different calculations. If we calculated the 50% chance, the first calculation would be the same as the second calculation so we could just add the first calculation to itself. If we are calculation 25% and 75% though we get both calculations and at the end add them together hence summation.

To calculate the entropy of a specific event X with probability P(X) you calculate this:

-P(X)*log2(P(X))

As an example, let's calculate the entropy of a fair coin.
The probability of heads is 50%. Here's the entropy we get when plugging that 0.5 into the equation:

Eheads=-0.5*log2(0.5)
Eheads=-0.5*-1
Eheads=0.5
Etails=0.5
E=0.5+0.5
E=1

Now lets plug in .25 and .75:

Etwenty-five=-0.25*log2(0.25)
Etwenty-five=-0.25*-2
Etwenty-five=0.5
Eseventy-five=-0.75*log2(0.75)
Eseventy-five=-0.75*-0.415
Eseventy-five=0.311
E=0.5+0.311
E=0.811
